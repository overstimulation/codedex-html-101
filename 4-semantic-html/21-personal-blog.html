HTML5 Semantic Elements Cheatsheet

Semantic elements clearly describe their meaning in a human- and machine-readable way. They help with accessibility, SEO, and code clarity.

Main Semantic Elements

  • <header>: Introductory content or navigation links for a section/page.
  • <nav>: Navigation links/menu.
  • <main>: Main content unique to the page (only one per page).
  • <section>: Thematic grouping of content, typically with a heading.
  • <article>: Independent, self-contained content (e.g., blog post, news article).
  • <aside>: Content tangentially related to the main content (e.g., sidebar, pull quote).
  • <footer>: Footer for a section/page (e.g., copyright, contact info).
  • <figure>: Self-contained content, often with an image, diagram, or code snippet.
  • <figcaption>: Caption for a <figure>.
  • <mark>: Highlighted or marked text.
  • <time>: Machine-readable date/time.
  • <address>: Contact information.

Example Structure

<body>
  <header>...</header>
  <nav>...</nav>
  <main>
    <article>
      <header>...</header>
      <section>...</section>
      <footer>...</footer>
    </article>
    <aside>...</aside>
  </main>
  <footer>...</footer>
</body>
